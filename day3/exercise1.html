Dynamic UI manipulation using data obtained via fetch

Answers to Questions:

What is a promise?

A promise is an object representing the eventual completion or failure of an asynchronous operation. It allows you to handle asynchronous operations more cleanly and synchronously in your code.

What does the fetch method return?

The fetch method returns a promise that resolves to the Response object representing the response to the request.

Why do we need to have two .then callbacks with our fetch code?

We use two .then callbacks with fetch code to handle asynchronous data retrieval and manipulation. The first .then callback converts the response to JSON format, and the second .then callback is used to process the data after it's available.

What can and should you do in the first .then?

In the first .then, you should handle the initial response from the server, which may include checking for errors and converting the response to a usable format, like JSON.

What can and should you do in the last .then?

In the last .then, you should handle the data that has been successfully retrieved and processed. You can update the DOM or perform any other actions based on the data received from the server.
